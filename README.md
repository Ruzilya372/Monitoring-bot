# Telegram-бот для мониторинга IT-мероприятий и хакатонов

R&D-проект (прототип) по парсингу RSS-потоков.

> **СТАТУС ПРОЕКТА:** Данный репозиторий представляет собой **исследовательский Pet-проект (Research / MVP)**. Логика полностью реализована для демонстрации работы пайплайна мониторинга сайтов.
Инструмент автоматического сбора, очистки и фильтрации анонсов студенческих мероприятий, хакатонов и летних школ из профильных RSS-лент с последующей отправкой целевых уведомлений в Telegram.

## Решаемая проблема
В огромном потоке публикаций на профильных IT-ресурсах легко пропустить важные анонсы соревнований и учебных программ для начинающих специалистов.

## Архитектура и эволюция проекта
Изначально планировалось собирать посты напрямую из профильных Telegram-каналов, так как основная масска объявления студенческий мероприятий происходит именно в Telegram, однако в ходе анализа документации было учтено жесткое ограничение платформы: *боты не могут мониторить чужие каналы, если они не являются в них администраторами*. 
Использование альтернативных библиотек (например, `Telethon` для UserBot) в рамках учебного MVP было неоправданным риском из-за сложной регистрации и высокой вероятности блокировки номера при отладке бесконечных циклов запросов. Поэтому в качестве тестового формата были выбраны **RSS-ленты Хабра** (разделы AI и Big Data).

Код спроектирован по принципу разделения ответственности (Single Responsibility) и разбит на 3 файла...
1. **ETL-слой (main.py):** Сбор данных из RSS-потоков через `feedparser` и очистка текста от HTML-тегов с помощью регулярных выражений.
2. **Конфигурация (config.py):** Изолированное хранение токенов доступа Telegram API и массива ключевых слов для безопасности и удобства настройки.
3. **Логический движок (main.py):** Фильтрация датафрейма Pandas и генерация кастомизированных Markdown-уведомлений в Telegram-канал.

## Стек технологий
* Python 3
* Pandas (обработка и фильтрация данных)
* Feedparser (парсинг RSS-лент)
* Requests (взаимодействие с Telegram API)
